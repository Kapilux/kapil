Workforce on Wheels

Enabling productivity in a connected car environment

1 months
Lead Product Designer
Car Infotainment

Overview

The "Workspace on Wheels" (WOW) is a PoC solution designed to transform the car into a functional and safe mobile workspace. Built through a collaboration between HARMAN and Salesforce, the experience leverages Salesforce’s Einstein Voice AI and HARMAN’s in-car connectivity platform to enable professionals to stay productive on the go without compromising safety.

My Role: I was part of the project as a Sr. UX designer and was responsible for UX/UI Design.

Collaboration: Business Stakeholder, Project Manager, and Dev team.

Tools: Sketch, Trello, MS Teams

Goals & Objectives

The objective of this project is to provide a digital interface i.e a mobile app where a user is able to operate various car controls. Features that are right now handled through a key or physical dashboard option available in the vehicle.

Strategic Goals
  • Showcase a futuristic in-car enterprise productivity system.
  • Extend Salesforce functionality into a new environment (automotive).
  • Reduce cognitive load and distractions while enabling useful business functions.
Key Use Cases
  • Voice-controlled account briefing and daily summaries.
  • Safe communication features like text dictation.
  • In-car lead generation using Salesforce AI.
  • Location-based contextual recommendations.
  • Monetization through car commerce and in-vehicle marketplace.

Discovery Phase

To align user needs with product goals, HARMAN’s design studio Huemen led a comprehensive discovery process.

Activities
  • User & Secondary Research
  • Salesforce + HARMAN Design Thinking Workshop
  • Opportunity Mapping
  • Documentation of insights and design direction
Key Insights
  • SMB users value contextual data over generic dashboards
  • Safety and low distraction UX are top priorities
  • Voice interaction is preferred when multitasking
  • Mobile professionals want seamless continuity across environments (office → car → field)
Research Outcomes

A Design Thinking Workshop identified synergy opportunities:

  • Integration of Salesforce features like lead generation, briefings, and note-taking.
  • Alignment with smart cities and manufacturing initiatives.
  • Identified new user segments such as sales reps and mobile managers.

Prioritized Features:

  • Lead Generation
  • Safe Communication
  • Account Briefing
  • Car Commerce Integration
  • Contextual Recommendations (location-based)
  • NLP-Driven Data Visualization
  • Authentication via Face Recognition
  • Account Summary & Updates
  • Subscription & Marketplace Model
Feature Mapping Workshop

As a pivotal step in aligning stakeholder goals with user needs, a Feature Mapping Workshop was conducted jointly by HARMAN and Stakehlders. The session aimed to identify, prioritize, and validate in-car productivity features that could be safely executed in a mobile environment. We used Trello as a tool for online collaboration.

Design Phase

Leveraging HARMAN’s Titan UI framework, the design team built a scalable system based on user journeys and Salesforce integration.

Deliverables

  • Moodboard & Design Language
  • Information Architecture
  • Voice Dialog Flow (for NLP/AI interactions)
  • Storyboarding
  • Click-through Prototypes
  • Visual Design Mockups

Information Architecture

The Information Architecture (IA) for Workspace on Wheels was designed to ensure seamless access, low cognitive load, and context-aware navigation in a driving environment. Built around voice-first interaction and glanceable UI, the IA structured all features to support fast, safe, and relevant user flows.

Voice Interaction Design

WOW emphasized Voice AI as the core interaction model to reduce distraction while driving. Salesforce's Einstein Voice enabled:

  • Natural conversation flow for daily briefings and CRM updates.
  • Quick response commands (e.g., “Get account summary,” “Find leads nearby”).
  • Integration of voice input with on-screen data visualization.

Wireframes

The wireframe phase translated early concepts into structured, low-fidelity screens that defined layout, content hierarchy, and user flow—setting the groundwork for a safe, intuitive in-car experience.

  • Visualize the end-to-end user journey across key touchpoints.
  • Validate task flows for primary use cases: briefings, lead generation, and communication.
  • Simplified layouts that supported glanceable content and voice-first interaction.
  • Ensure layout and interaction patterns followed driver safety guidelines

Visual Design

The Visual Design phase brought the functional wireframes to life, transforming core workflows into an engaging, brand-aligned, and safety-first interface tailored for in-car use.

  • Apply a clean, bold visual language optimized for quick glances and minimal interaction.
  • Ensure consistency with Salesforce’s brand while adapting for automotive environments.
  • Design with accessibility and safety as core tenets—large typography, high contrast, voice-first affordances.
  • Dark mode interface to reduce screen glare during driving.
  • Seamless integration of voice interaction cues and confirmation feedback.

Impact

The visual design successfully merged enterprise productivity tools with automotive UX standards, creating a refined experience that is both functional and emotionally resonant—helping users stay connected without distraction.

Conclusion

From initial research and user insights to wireframes and high-fidelity visual design, every phase was driven by the goal of balancing enterprise utility with driver safety. By leveraging contextual intelligence, intuitive interaction models, and seamless UX flows, the final concept empowers users to manage leads, meetings, and communication without compromising their attention on the road.
