Car Control App

Digital mobile app to control car systems

1 months
Sr. UX Designer
Mobile

Overview

The future of mobility lies in digital integration. With rapid advancements in automotive and mobile technology, users expect seamless, smart experiences. This project aims to eliminate the dependency on physical car keys by enabling a mobile application that allows users to interact with and control various in-car functions—simplifying vehicle access and enhancing the overall driving experience.

My Role: I was part of the project as a Sr. UX designer and was responsible for UX/UI Design.

Collaboration: Business Stakeholder, Project Manager, and Dev team.

Tools: Sketch, Jira, MS Teams

Design Process

The objective of this project is to provide a digital interface i.e a mobile app where a user is able to operate various car controls. Features that are right now handled through a key or physical dashboard option available in the vehicle.

Goals

User Objective
  • Achieve a seamless, enjoyable interaction with their vehicle.
  • Access a safe and easy-to-use interface.
Business Objectives
  • Deliver a delightful and secure digital experience.
  • Strengthen customer loyalty through convenience.
  • Build brand trust and credibility.

Research & Insights

Industry Context

The automotive sector has shifted dramatically with the advent of EVs, connected vehicles, and autonomous tech. Users are increasingly interacting with their environment via mobile apps—from smart homes to connected mobility.

Observed User Behavior

TUsers are overwhelmed with complex physical dashboards, multiple buttons, and the constant need to carry physical keys. Modern users expect:

  • Minimal, touch-first experiences.
  • Mobility solutions that align with smartphone usage patterns.
Feature Mapping

We started with identifying key features aligned to usage.

Feature Categorization

We need to sort out features in groups for better understanding and their usage. We are grouping our features in three different categories as per their usage. First category is for features that are related to outside environment, second is features that are mostly used inside the car and the third one is to provide vital information to user.

Features grouped by usage context:

  • External Controls: Unlock, Engine Start, Trunk Access.
  • In-Car Controls: Doors, Climate, Lights.
  • Information & Utilities: Fuel status, Navigation, Profile, History.

Key Problems Identified

Features were never part of problems when it comes to usage but bad UX and UI are the main reasons for various design failures. Physical interfaces are more complicated and sometimes create chaos while using. Alternative to these screens and solutions is Mobile App. So, how we are going to provide a safer, preferable and easy to use solution to our user on their smartphone.

  • Dependency on physical keys.
  • Overloaded car dashboards causing confusion.
  • Unclear feedback and information architecture.
  • Lack of user-friendly digital alternatives.
  • Increased cognitive load due to unintuitive interfaces.

Ideation

Core Design Approach

Leverage the ubiquity of smartphones to transform how users interact with their cars. Key design principles:

  • Prioritize usability and clarity.
  • Present only the most relevant features upfront.
  • Use familiar, intuitive icons and gestures.
  • Create a calm and minimal visual hierarchy.

Task Flow

A defined task flow ensures a consistent user journey:

Paper Prototype

Before diving into digital design, paper prototyping allowed us to:

  • Sketch initial layout ideas quickly.
  • Test navigation logic with users.
  • Validate feature prioritization and interaction simplicity.
  • Explore multiple variations of key flows like unlocking, starting the engine, and navigating to feature categories.
Key Learning:

Users appreciated minimal interfaces and requested quicker access to daily-use functions (unlock, start, AC). This helped solidify our decision to surface key actions on the home screen.

Wireframes & Design Opportunities

Wireframes provided a mid-fidelity layout to visualize and iterate on UI structure, before applying visual design.

Opportunities Discovered:
  • Home Screen Optimization: Users wanted to perform top tasks without scrolling—so we prioritized engine start, unlock, and trunk on the landing screen.
  • Information Transparency: Users needed feedback during operations (e.g., car starting, door locked). Microinteractions were planned for these.
  • Feature Discoverability: Users felt overwhelmed by options in other car apps. Grouping features in logical sections reduced cognitive load.
  • Fail-safes: Bluetooth disconnection states and fallback prompts were introduced for seamless recovery.

Visual Design

The visual design phase translated the functional wireframes into an aesthetically pleasing, intuitive user interface that aligned with modern mobility trends. The focus was on clarity, minimalism, and ease of interaction, especially in high-stress or on-the-go situations. Each screen was crafted to reduce cognitive load while delivering essential functionality through a clean, icon-driven interface.

Onboarding Screen

Branding splash screen (conceptually using BMW logo).

Connection Screen

Bluetooth pairing prompt with fallback messaging.

Unlock Screen

Secure swipe gesture to prevent accidental access.

Home Screen

Three core actions: Start Engine, Unlock Car, Open Trunk.

Start Engine

Visual feedback loop showing ignition progress.

Dashboard

Aggregated status + quick access to key features like fuel range and AC.

Doors

Visual status indicators for each door.

Lights

Manual and auto brightness adjustments.

Climate

Auto and manual controls. Outside temperature display.

Location

Input destination and receive directions.

Controls

Secondary features with quick-glance status.

Profile

Edit profile, view service history, and referral options.

Outcome

The design successfully transforms the physical car key experience into a digital interaction. It minimizes complexity, enhances usability, and fits into users’ mobile-first lifestyles. Prioritizing ease-of-use, feedback, and security makes the app a compelling companion for car owners.

Conclusion

This car control app builds a bridge between user needs and technological possibilities. By replacing the physical key with a thoughtfully designed mobile interface, it not only improves convenience but strengthens trust in modern automotive UX.
